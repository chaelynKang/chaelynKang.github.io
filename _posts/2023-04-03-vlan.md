# VLAN

* VLAN(Virtual Local Area Network)은 물리적으로 서로 다른 네트워크 세그먼트에 위치하더라도 동일한 물리적 네트워크에 있는 것처럼 서로 통신할 수 있는 네트워크 디바이스의 논리적 그룹이다.

* 

VLAN(Virtual Local Area Network)은 물리적으로 서로 다른 네트워크 세그먼트에 위치하더라도 동일한 물리적 네트워크에 있는 것처럼 서로 통신할 수 있는 네트워크 디바이스의 논리적 그룹입니다. VLAN은 일반적으로 대규모 네트워크에서 네트워크 성능, 보안 및 관리 용이성을 향상시키는 데 사용됩니다.

VLAN은 OSI 모델의 데이터 링크 계층(레이어 2)에서 작동하며 스위치를 사용하여 구현됩니다. 스위치가 VLAN으로 구성된 경우 스위치의 물리적 포트를 논리적으로 서로 다른 VLAN으로 분리할 수 있으며 각 VLAN은 고유한 IP 서브넷 및 네트워크 주소를 가질 수 있습니다. 이를 통해 네트워크 관리자는 부서, 기능 또는 보안 요구 사항에 따라 네트워크를 세분화하면서도 서로 다른 VLAN에 있는 장치 간의 통신을 허용할 수 있습니다.

VLAN의 이점 중 일부는 다음과 같습니다:

네트워크 성능 향상: VLAN은 네트워크의 브로드캐스트 트래픽을 줄여 네트워크 성능을 향상시키고 네트워크 정체 가능성을 줄일 수 있습니다.

향상된 네트워크 보안: VLAN을 사용하여 중요한 데이터 또는 애플리케이션을 격리하여 무단 액세스를 방지하고 네트워크 공격 위험을 줄일 수 있습니다.

네트워크 관리 간소화: 관리자가 서로 다른 VLAN을 개별적으로 구성하고 모니터링할 수 있으므로 VLAN을 사용하면 네트워크를 보다 쉽게 관리할 수 있습니다.

VLAN에는 다음과 같은 몇 가지 유형이 있습니다:

포트 기반 VLAN: 포트 기반 VLAN에서는 스위치의 각 물리적 포트가 특정 VLAN에 할당됩니다.

태그가 지정된 VLAN: 태그가 지정된 VLAN에서는 패킷이 속한 VLAN을 식별하는 VLAN 식별자로 태그가 지정됩니다. 따라서 스위치의 동일한 물리적 포트가 여러 VLAN의 트래픽을 전송할 수 있습니다.

프로토콜 기반 VLAN: 프로토콜 기반 VLAN에서 패킷은 패킷 헤더에 사용되는 프로토콜에 따라 VLAN에 할당됩니다.

요약하자면, VLAN은 대규모 네트워크를 세분화하고, 네트워크 성능 및 보안을 개선하며, 네트워크 관리를 단순화하는 유연하고 확장 가능한 방법을 제공합니다.
